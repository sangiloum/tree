## Project Overview

Interactive D3.js visualization of the academic advisor/student genealogy for members of the [IBS Discrete Mathematics Group (DIMAG)](https://dimag.ibs.re.kr). Data is scraped from the [Mathematics Genealogy Project](https://www.mathgenealogy.org).

## Development Commands

```bash
# Serve locally (no build step needed — fully static)
python -m http.server 8000
# Open http://localhost:8000

# Install Python scraper dependencies
cd scripts && pip install -r requirements.txt

# Regenerate genealogy.json from mathgenealogy.org (~10–30 min)
cd scripts && python scrape_genealogy.py

# Download/update profile photos from DIMAG site and Wikipedia
python scripts/download_photos.py
```

## Architecture

This is a single-page static app with no build tooling or bundler.

- **`index.html`** — app shell; loads D3.js v7 from CDN and `main.js`
- **`main.js`** — all visualization logic: D3 force simulation, zoom/pan, sidebar, search, filters, depth slider
- **`style.css`** — dark cosmic theme
- **`data/members.json`** — manually maintained list of DIMAG members with their MGP IDs, roles (`current`/`former`), `photo_url`, and `left_year`
- **`data/genealogy.json`** — generated by `scripts/scrape_genealogy.py`; commit after regenerating
- **`data/wikipedia_urls.json`** — generated by `download_photos.py`; maps MGP ID → Wikipedia URL; used by `main.js` for sidebar links
- **`data/photos/`** — profile photos stored as `{mgp_id}.jpg`, fetched from the DIMAG site and Wikipedia
- **`scripts/scrape_genealogy.py`** — BFS crawler that reads `members.json` as seeds and writes `genealogy.json`
- **`scripts/download_photos.py`** — fetches member photos from `dimag.ibs.re.kr` and ancestor photos from the Wikipedia API; also updates `wikipedia_urls.json`

## Data Flow

1. `members.json` seeds the scraper with DIMAG member MGP IDs
2. `scrape_genealogy.py` crawls mathgenealogy.org BFS upward (unlimited depth by default) and downward (3 generations) from seeds, producing `genealogy.json`
3. `main.js` fetches both JSON files on load, builds the D3 force graph

## Key Layout Logic (`main.js`)

- Year axis is horizontal: `yearToWorld(year)` maps PhD year → X position (older = further right in world space, which appears as left/top after zoom transform)
- Nodes are typed: `current`, `former`, `ancestor`, `ancestor_deep` — controls color, radius, and glow filter
- LOD: labels and deep-ancestor nodes are hidden below zoom thresholds `LABEL_HIDE_ZOOM` (0.22) and `NODE_HIDE_ZOOM` (0.12)
- Double-click on an ancestor activates lineage filter (`isFiltered`, `filteredNodeIds`) highlighting the path down to DIMAG members

## Updating Member Data

Edit `data/members.json` — each entry needs `mgp_id` (integer or `null`), `name`, and `role` (`"current"` or `"former"`). Then re-run the scraper.

## Scraper Configuration

Edit constants at the top of `scripts/scrape_genealogy.py`:
- `ANCESTOR_DEPTH` — max generations upward (`None` = unlimited)
- `DESCENDANT_DEPTH` — max generations downward from seeds (default: 3)
- `SLEEP_BETWEEN_REQUESTS` — seconds between HTTP requests (default: 1.5; keep polite)

## Deployment

GitHub Pages: push to `main`, enable Pages with root `/`. No build step needed.
